# Checklists for Prompt Engineering

Набор чек-листов для проверки качества промптов, workflow и безопасности.

---

## 1. Чек-лист «Перед отправкой промпта»

1. **Цель понятна?**
   - Я чётко сформулировал, что хочу получить (результат, а не процесс).

2. **Роль модели задана?**
   - Есть фраза «Ты – [роль]…».

3. **Есть контекст?**
   - Есть минимум 1–3 предложения о ситуации, аудитории, ограничениях.

4. **Задача разбита на пункты?**
   - Основные действия вынесены в нумерованный или маркерный список.

5. **Формат ответа указан?**
   - Список / таблица / JSON / эссе / план – явно прописано.

6. **Есть ограничения?**
   - Указано, чего делать не надо (например, «не придумывай факты»).

7. **Проверка результата?**
   - В конце промпта есть фраза типа: «В конце кратко проверь, что все пункты выполнены».

---

## 2. Чек-лист «Дизайн workflow / агента»

1. **Вход и выход определены?**
   - Что принимает агент на вход?
   - Что должен вернуть (текст, файл, несколько вариантов)?

2. **Шаги понятны?**
   - Workflow разбит на шаги 1–2–3–4…
   - Для каждого шага понятно: что делает агент, что делает человек.

3. **Инструменты обозначены?**
   - Там, где нужен Browse, Code Interpreter, Actions – это явно указано.

4. **Ошибки и ограничения продуманы?**
   - Понимаем, что делать, если данных не хватает;
   - Предусмотрен «безопасный отказ».

5. **Тестовые кейсы есть?**
   - Придуманы 1–3 тестовых сценария для проверки workflow.

---

## 3. Чек-лист «Безопасность knowledge-файлов»

1. Я не загружаю:
   - персональные данные,
   - коммерческие тайны,
   - финансовые пароли, ключи и т.п.

2. Каждый файл:
   - имеет понятное название;
   - не содержит «сырой» приватной информации;
   - описывает только то, что нужно модели.

3. Я понимаю, что:
   - модель видит всё содержимое файла;
   - всё, что загружено, потенциально может появиться в ответе.

4. Для конфиденциальных сценариев:
   - использую обезличенные данные или синтетические примеры.
