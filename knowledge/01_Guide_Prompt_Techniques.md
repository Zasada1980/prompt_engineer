# Guide to Prompt Engineering Techniques

Этот файл – основной справочник по техникам промт-инжиниринга для агента «Промт-Инженер».

## 1. Базовые принципы

1. **Структура WHO/WHAT/WHY/HOW**
   - WHO – какую роль играет модель (кто она?).
   - WHAT – что именно нужно сделать.
   - WHY – зачем это нужно (контекст, цель).
   - HOW – как оформлять ответ (шаги, формат, ограничения).

2. **Явные делимитеры**
   Используйте разделители: тройные кавычки, Markdown-кодблоки, метки `<Section>` для отделения частей промпта:
   ```text
   Контекст:
   """ ... """

   Задача:
   """ ... """
   ```

3. **Позитивные формулировки**
   Вместо «не пиши лишний текст» – «будь краток, отвечай по пунктам».

4. **Проверка результата**
   Заканчивайте промпт инструкцией:
   * «проверь, что ответ соответствует всем пунктам задания»;
   * «в конце дай краткий чек-лист того, что сделано».

## 2. Часто используемые техники

### 2.1. Few-Shot Prompting

Суть: показываем 1–3 примера задач и правильных ответов, а затем даём новую задачу без ответа.

**Шаблон:**

```text
Ты – [роль].

Примеры:

Вход:
"""[пример входа 1]"""
Выход:
"""[пример ответа 1]"""

Вход:
"""[пример входа 2]"""
Выход:
"""[пример ответа 2]"""

Теперь обработай новый вход по той же схеме:

Вход:
"""[новый ввод]"""
Выход:
```

### 2.2. Ролевая постановка (Role Prompting)

Задаём модели конкретную роль:
* «Ты – преподаватель…»
* «Ты – редактор текста…»
* «Ты – инженер по данным…»

Это помогает модели выбирать нужный стиль, глубину и формат.

### 2.3. RAG (Retrieval-Augmented Generation)

Суть: перед генерацией ответа подгружаем релевантные данные (knowledge-файлы, документы) и явно говорим модели:
* «Используй только эти источники»,
* «Если информации не хватает, честно скажи об этом».

### 2.4. ReAct

Сочетание рассуждения и действий:
* модель размышляет: «Reasoning: …»
* затем вызывает действие: «Action: …»
* затем интерпретирует результат.

В классическом ReAct это делается пошагово; в нашем агенте достаточно понимать идею: сначала план, потом действия.

## 3. Общий шаблон «хорошего промпта»

```text
Роль: [кто ты?]

Контекст:
"""[краткое описание ситуации]"""

Задача:
- [пункт 1]
- [пункт 2]

Формат ответа:
- [список/таблица/JSON]
- [краткость/детальность]

Ограничения:
- [что не делать/какие темы не трогать]

В конце:
- проверь, что выполнил все пункты;
- кратко перечисли, что именно ты сделал.
```

## 4. Когда применять какую технику

* **Few-Shot** – когда важен стиль или формат результата.
* **Role Prompting** – почти всегда, чтобы зафиксировать «персону» модели.
* **RAG** – когда надо опираться на конкретные документы или знания.
* **ReAct-подход** – при сложных многошаговых задачах (план + действия).
